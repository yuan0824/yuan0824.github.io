linux-FTP与SSH

发表于 2019-08-06

在操作远程linux系统的时候，经常会使用SSH协议登录，执行linux命令；有时也需要FTP协议登录，进行大文件传输，结合使用可以满足大部分需求。

操作环境

在CentOS 7的环境下进行测试

  • OS：CentOS 7 x64
  • RAM：1024 MB
  • Storage：25 GB SSD

FTP

FTP（文件传输协议）是一个用于在计算机网络上在客户端和服务器之间进行文件传输的应用层协议。

端口

默认运行在20和21两个端口

  • 20：在客户端和服务器之间传输数据流
  • 21：传输控制流

作用

可靠和高效的传输数据，主要用来传输文件

Linux服务器端工具

  • vsftpd是一个较为安全的FTP服务器软件，安装命令yum install vsftpd
  • vsftpd默认设置为禁止root用户登录ftp，需要新建用户

Window客户端工具

WinSCP是Windows系统下开源的FTP客户端，支持SFTP，FTP，SCP等传输

SSH

SSH（安全外壳协议）是建立在应用层基础上的网络传输协议。

端口

默认端口22

作用

客户端与远程主机交互，主要用来执行linux命令

Linux服务器端

OpenSSH是主流的SSH协议的免费开源实现

Window客户端工具

Xshall是一款强大的安全终端模拟软件，有免费的家庭/校园版，支持SSH，TELNET等交互方式。

用户命令

推荐使用非root用户登录
