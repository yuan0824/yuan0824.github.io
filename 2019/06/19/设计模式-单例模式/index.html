设计模式-单例模式

发表于 2019-06-19

单例模式（Singleton Pattern）属于创建型模式，只涉及到一个单一的类，该类负责创建自己的对象，同时确保只有单个对象被创建。这个类提供了一种访问其唯一的对象的方式，可以直接访问，不需要实例化该类的对象。

特性

  • 该类只有一个实例
  • 该类必须自己创建自己的实例
  • 该类创建的实例被其他所有对象所共享

优点

  • 内存中只有一个实例，减小了内存的开销
  • 避免对共享资源过多的占用

使用场景

  • 与数据库建立连接
  • 对配置文件的读取
  • 网站计数器
  • Windows的回收站和资源管理器
  • Spring bean对象

构建方式

分为两种构造方式，懒汉式和饿汉式

饿汉式

立即加载，全局的单例实例在类装载时构建

懒汉式

懒加载，全局的单例实例在第一次被使用时构建

DCL(双重检查锁)

INSTANCE = new Singleton();具体实现大致分为三个步骤

  1. 分配对象内存空间（new指令）
  2. 初始化对象（invokespecial指令）
  3. 设置instance指向分配的内存地址（putstatic指令）

可能存在指令重排，导致1-3-2的情况出现，导致错误，所以需要使用volatile关键字来禁止指令重排，保证正确性。

静态内部类

使用DCL过于繁琐，也可以使用静态内部类实现
